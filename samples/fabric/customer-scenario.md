# Sample customer scenario — Microsoft Fabric (generic)

## Customer

Generic enterprise customer (anonymized).

## Industry

Cross-industry (suitable as a generic Fabric sample).

## Region

Global.

## Audience

CDO, data platform leads, data engineers, analytics engineers, BI developers,
business analysts.

## Microsoft product focus

Microsoft Fabric — OneLake, Lakehouse, Pipelines, Notebooks, Semantic Models,
Power BI, Real-Time Intelligence.

## Business challenges

- Fragmented data estate across multiple platforms and tools.
- Slow, brittle ETL pipelines.
- Inconsistent semantic layer leading to mismatched KPIs.
- Long time-to-insight for business users.

## Workshop objective

Show how Microsoft Fabric unifies data engineering, data science, real-time,
and BI on a single SaaS platform anchored on OneLake.

## Requirements

| ID | Requirement |
|----|-------------|
| R1 | Demonstrate ingesting data into a Lakehouse via a Pipeline. |
| R2 | Show transformation in a Notebook (PySpark) with Delta tables. |
| R3 | Show building a Semantic Model and a Power BI report on top. |
| R4 | Show a real-time event flow into an Eventhouse (KQL). |
| R5 | Show governance: workspace roles, sensitivity labels, lineage. |

## Success criteria

- Data leaders see a unified estate story.
- Engineers leave with a pattern for Bronze/Silver/Gold on OneLake.
- BI developers see a consistent semantic layer.

## Executive summary (one sentence)

Microsoft Fabric collapses the data stack into one governed SaaS platform so
your teams can move from raw data to decision in hours, not weeks.
