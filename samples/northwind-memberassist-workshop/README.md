# Northwind MemberAssist — worked workshop sample

A complete, named, customer-flavoured example of what a workshop generated by
this accelerator should look like *after* customization.

> **Domain:** US health-plan member services
> **Customer (fictional):** Northwind Health
> **Audience:** member-services leaders, claims operations, plan architects, compliance
> **Format:** half-day, 9 agenda items
> **Stack:** FastAPI 0.115 · Jinja2 · Vanilla JS · CSS-only design system · Docker
> **Demos:** chat (benefits Q&A), search (provider directory), workflow (claim status), document analysis (EOB extraction), evaluation (with PHI-leak gauge)

This folder is read-only documentation: it is what a CSA would produce *and
ship to the customer* after running the tutorial. Every file here is filled
in — no `[brackets]`. Use it as the gold reference when retargeting to your
own engagement.

## What's in this folder

| File / folder | What it is |
|---|---|
| `customer-scenario.md` | Filled scenario: 6 requirements, 5 success criteria, 4 personas. |
| `.github/skills/workshop-creation/SKILL.md` | Customized SKILL with frontmatter, located the way the reference architecture expects. |
| `agenda.md` | 9 items under the heading anchor parsed by `agenda_loader.py`. |
| `prompts.md` | Six Copilot prompts: `/plan`, implement, customize, three push-backs. |
| `data/*.json` | Realistic mock data for the five demos. |
| `sections/_section-template.md` | Annotated drop-in template for any explanatory section. |
| `sections/demo-3-claim-status.md` | One section fully written end-to-end as the gold sample. |
| `presenter-notes.md` | One block of presenter notes per agenda item. |
| `customization-diff.md` | What changed vs. `samples/generic-ai-workshop/` and why. |

## How to use it

1. Read it top-to-bottom once before your first engagement.
2. Fork it into a new repo (e.g. `contoso-rxassist-workshop`) and rename
   "Northwind / MemberAssist / claim / EOB" to your customer's vocabulary.
3. Keep `customization-diff.md` updated as you make changes — the next CSA
   will thank you.

## What it deliberately does *not* include

- No real Foundry / Azure SDK calls. Mocks only. The real-Azure path is
  documented in module 9 (`docs/09-interactive-demos.md`) under "Advanced
  pattern: real-Azure extension" using the `infra/scripts/*.ipynb` pattern.
- No PHI. All names, member IDs, claim IDs, and provider data are fictional.

## Worked example, not a runnable app

This sample is the *content* (scenario, SKILL, agenda, prompts, data,
sections, notes) — not the FastAPI app itself. The app is generated by
following the tutorial with these inputs. See `docs/14-worked-example.md`
for the read-only walkthrough.
