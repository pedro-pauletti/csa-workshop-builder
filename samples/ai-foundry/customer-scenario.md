# Sample customer scenario — Microsoft AI Foundry (generic)

## Customer

Generic enterprise customer (anonymized).

## Industry

Cross-industry (suitable as a generic Foundry sample).

## Region

Global.

## Audience

CIO/CTO, enterprise architects, AI/ML engineers, application developers,
security architects.

## Microsoft product focus

Azure AI Foundry — agents, model catalog, evaluations, tracing, prompt flow.

## Business challenges

- Multiple AI proofs-of-concept that never reach production.
- No consistent evaluation discipline across AI use cases.
- Unclear governance, observability, and cost story for AI workloads.
- Fragmented model and tool selection across teams.

## Workshop objective

Show how Azure AI Foundry provides a unified path from prototype to production
for enterprise AI agents — with built-in evaluation, tracing, and governance.

## Requirements

| ID | Requirement |
|----|-------------|
| R1 | Demonstrate building a grounded AI agent against a curated knowledge source. |
| R2 | Show how to evaluate agent quality with reproducible test sets. |
| R3 | Show end-to-end traces for a multi-step agent invocation. |
| R4 | Show model selection and routing across the catalog. |
| R5 | Show governance: prompts, datasets, evaluations versioned together. |

## Success criteria

- Architects can describe the Foundry building blocks unprompted.
- Engineers leave with a runnable starter project pattern.
- Security/governance stakeholders see how their controls map to Foundry.

## Executive summary (one sentence)

Azure AI Foundry is the path from "interesting AI prototype" to a governed,
evaluated, production-grade agent your business can rely on.
