Semantic ranker

A second-stage re-ranker that uses a deep-learning model to reorder the top results by meaning, and produces captions and an extractive answer — the building blocks of high-quality RAG.

What the semantic ranker does

Azure AI Search executes your hybrid query, takes the top 50 results, and feeds them — together with the original question — to a multilingual deep model. The model re-scores each document on a 0–4 semantic scale and produces:

  • Semantic captions — the snippet most relevant to the question, highlighted.
  • Semantic answers — when the model is confident, an extractive answer pulled verbatim from a document.
  • Reordered @search.rerankerScore — replaces the original blended score for the top results.
Cost note. Semantic ranker is a billable feature at the service tier. Use it on user-facing queries, not on bulk pipelines, and turn it off in load tests until you have a real evaluation harness.

When it changes ranking

The biggest gains come from queries where the keyword and the answer are paraphrases of each other, or where the most relevant document is one the BM25 + vector blend buried at rank 8–15. The example below is exactly that case.

Two-stage ranking pipeline

L1 finds 50 plausible candidates fast and cheap; L2 reorders them with a much more expensive model that has seen the query and the documents.
