Evaluation & next steps

A RAG system you cannot measure is a RAG system you cannot improve. Define metrics, a test set, and a delivery roadmap before you ship.

The four metrics every grounded-AI system needs

  • Groundedness — does the answer use only information present in the retrieved documents? Measured with an LLM-as-judge or with token-overlap heuristics.
  • Relevance — is the answer responsive to the question?
  • Retrieval recall@k — does the top-k from Azure AI Search contain at least one document that can answer the question? This is the lever you tune when groundedness is low.
  • Latency p95 — end-to-end milliseconds for the full retrieve → rerank → generate loop. Owns user perception.
Microsoft's evaluation tooling. Use the Azure AI Foundry evaluation SDK to run all four metrics against a fixed test set and graph the trend across runs. It plugs directly into Azure AI Search retrieval.
