Grounded chat (RAG)

Combine Azure AI Search retrieval with an LLM and you get a chat that answers only from your content, with citations. This is the canonical RAG pattern.

The RAG loop in one diagram

For every user turn, the application:

  1. Embeds the question with the same model used at index time.
  2. Runs a hybrid + semantic query against Azure AI Search.
  3. Takes the top N documents (typically 3–8) and passes them, with the question, to the LLM as context.
  4. Instructs the LLM (in the system prompt) to answer only from the provided context and to cite each document by its index identifier.
  5. Streams the answer to the user with inline citation links back to the source.

Why the system prompt matters

The single biggest lever on answer quality is the system prompt. It tells the LLM what tone to use, what to refuse, and — crucially — to say "I don't know" when retrieval comes back empty. Without that instruction the model will hallucinate confidently from its pre-training data.

Microsoft's reference implementation. The azure-search-openai-demo repository is the canonical end-to-end RAG example — Azure AI Search + Azure OpenAI + a chat web app. Read its app.py and approaches/ folder before designing your customer's solution.
